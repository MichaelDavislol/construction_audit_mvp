# Руководство пользователя

## Что подготовить

Для аудита существующей сметы:

- один план в `PNG`, `JPG` или `JPEG`;
- одну XLSX-смету;
- необязательно — один ZIP с 1–5 фотографиями объекта.

Для предварительной сметы достаточно одного плана. Чем яснее подписи и размеры на плане, тем меньше уточнений потребуется.

## Сценарий: план и готовая смета

1. Загрузите план и XLSX и попросите провести предварительный аудит.
2. Скилл проверит файлы и передаст только план отдельной Vision-задаче.
3. Вы получите полный обзор распознанной геометрии.
4. Если есть ошибка, напишите точное исправление, например: «Высота всех окон 1,6 м».
5. Скилл создаст новую ревизию и снова покажет полный обзор.
6. Когда все данные верны, отдельным сообщением подтвердите геометрию.
7. Скилл получит каталог MCP, сопоставит позиции и выполнит расчёты.
8. Выберите: приложить ZIP с фотографиями или отдельно написать «без фото».
9. После аналитического этапа скилл сформирует итоговую сводку и HTML-отчёт.
10. После завершения аудита скилл предложит сформировать эталонную (контрольную) XLSX-смету по подтверждённой геометрии и ценам MCP.
11. Подтвердите генерацию отдельным сообщением. Без явного согласия файл не создаётся.

Исправление и подтверждение нельзя объединить в один шаг. После любого исправления нужно проверить новую ревизию и подтвердить её отдельным сообщением.

## Сценарий: только план

1. Загрузите один план и попросите подготовить предварительную смету.
2. Проверьте и при необходимости исправьте распознанную геометрию.
3. Подтвердите актуальную ревизию отдельным сообщением.
4. Скилл получит каталог цен и предложит сформировать XLSX.
5. Отдельно согласитесь на генерацию.

Если данных недостаточно, зависимые работы будут пропущены с указанием причины. Скилл не подставляет стандартную высоту потолка или другие типовые размеры.

## Что проверяется

При наличии нужной геометрии скилл рассчитывает контрольные значения для:

- грунтовки и окраски стен;
- устройства пола;
- отделки потолка;
- монтажа плинтуса;
- установки дверей;
- установки окон.

Межкомнатная дверь может присутствовать в геометрии двух помещений, но при подсчёте установки учитывается один раз по уникальному `element_id`.

## Как читать результат

Используются осторожные формулировки:

- «контрольное расчётное значение» — значение, рассчитанное из подтверждённой геометрии;
- «предварительное расхождение» — отличие от сметы выше выбранного допустимого отклонения;
- «строка требует проверки специалистом» — данных или однозначного соответствия недостаточно.

Наблюдения по фотографиям и аналитические гипотезы не заменяют расхождения, полученные детерминированным расчётом. Фотография показывает только конкретный кадр и не доказывает объём работ на всём объекте.

## Артефакты

Для каждого `job_id` Ouroboros хранит отдельный каталог состояния. В зависимости от ветки могут появиться:

- проверенные копии входных файлов;
- нормализованная смета;
- все ревизии геометрии и история исправлений;
- сохранённый каталог MCP;
- сопоставление строк, контрольные количества и трассировка расчётов;
- проверки цен, найденные расхождения и полнота проверки;
- результаты анализа фотографий;
- валидированные аналитические гипотезы;
- `report.html`;
- `generated_estimate.xlsx` — эталонная (контрольная) смета по подтверждённому плану и каталогу MCP — и описание пропущенных строк.

Скилл выводит путь к отчёту, но не открывает его автоматически без явной просьбы.

## Важные ограничения

- результат предварительный и требует профессиональной проверки;
- отсутствие найденных расхождений не доказывает отсутствие проблемы;
- неподдерживаемые работы остаются предупреждениями;
- валюта не предполагается, если MCP её не сообщил;
- фотографии не привязываются автоматически к помещениям;
- неясное исправление потребует уточнения, а не догадки.
