# Словарь терминов Construction Audit MVP

## Как пользоваться словарём

Здесь собраны пользовательские, строительные и технические термины, которые встречаются в скилле, трассировке и артефактах. Определения описывают именно смысл внутри Construction Audit MVP.

## A–Z и технические названия

### A2A

Agent-to-Agent — передача отдельной ограниченной задачи от основной LLM другому model-subagent. В скилле так запускаются Vision, Mapping, Photo Vision и Analyst.

### Analyst / аналитический subagent

Отдельная модельная задача после детерминированного аудита. Ищет системные связи и формулирует проверяемые гипотезы. Не может менять findings, расчёты или фото-наблюдения.

### Artifact / артефакт

Сохранённый результат этапа: JSON, XLSX, HTML или копия входного файла. Артефакт позволяет проверить, что именно участвовало в следующем шаге.

### Attachment / вложение

Файл, загруженный пользователем в сообщение. До импорта считается внешним входом и проходит проверку.

### Calculation trace / трассировка расчётов

Запись формулы, входных значений, результата, округления, помещения и предупреждений. Хранится в `calculation_trace.json`.

### Canonical / канонический

Приведённый к внутреннему однозначному виду. Например, видимое имя комнаты связывается со стабильным `room_001`, а работа — с `mcp_work_id`.

### Chat ID

Идентификатор диалога. Объединяет несколько пользовательских сообщений и несколько задач основной модели.

### Confidence

Оценка уверенности Vision в конкретном измерении. Не заменяет доказательство и не отменяет пользовательское подтверждение.

### Context package

Ограниченный сохранённый набор фактов, который получает Analyst. Он не видит произвольные файлы и не должен вводить данные вне context.

### Correction

Структурированное применение явного исправления пользователя к rooms, doors или windows. Correction не даёт основной LLM редактировать весь geometry JSON.

### Coverage / покрытие

Показатель полноты: сколько строк или цен проверено, сколько не проверено и почему. Нужен, чтобы частичный результат не выглядел полным.

### Decimal

Тип точной десятичной арифметики Python. Используется для строительных объёмов, цен и процентов вместо свободного счёта LLM.

### Delegation packet

Готовое задание для subagent: роль, цель, ожидаемый JSON, context, ограничения, model lane и режим доступа. Основная LLM должна передавать его без творческого пересказа.

### Deterministic / детерминированный

Выполняемый по фиксированному алгоритму. При одинаковом входе даёт одинаковый результат. В скилле расчёты, exact mapping и валидаторы детерминированы.

### Evidence / доказательство

Ссылка на конкретный источник значения или вывода: подпись на плане, строку сметы, finding, price check, trace или visual insight.

### Evidence ref

Структурированная ссылка аналитической гипотезы на существующий элемент context. Python проверяет, что такая ссылка действительно существует.

### Exact mapping

Сопоставление, которое Python строит без модели, когда названия и единицы совпадают точно и однозначно.

### Fail closed

Принцип «при недостатке или конфликте данных не угадывать». Система останавливает опасный этап либо помечает отдельную проверку как skipped/not checked.

### Finding

Предварительное расхождение, созданное Python-кодом. Например, количество ниже контроля или цена отличается от MCP. Finding не равен доказанному нарушению.

### Geometry / геометрия

Структурированное описание помещений, размеров, площадей, высот, дверей, окон, источников и предупреждений.

### Geometry review

Полная проверочная таблица геометрии, которую пользователь видит до подтверждения и расчётов.

### Handoff

Переход результата от subagent обратно основной задаче через runtime. Может сопровождаться системным напоминанием и скрытым промежуточным draft.

### Handoff barrier

Специальная остановка между сохранением результата subagent и показом пользовательского Markdown. После барьера renderer заново берёт данные из сохранённого состояния.

### Hash

Короткий цифровой отпечаток данных. В скилле используется SHA-256.

### HTML escaping

Преобразование специальных символов перед вставкой текста в HTML. Не позволяет пользовательскому или внешнему тексту стать исполняемой разметкой.

### ID

Стабильный идентификатор сущности. В скилле используются chat ID, task ID, job ID, room ID, photo ID, element ID, finding ID, insight ID и trace ID.

### Insight

Проверенная Python-валидатором аналитическая гипотеза. Содержит observation, hypothesis, limitations, evidence refs и recommended check.

### Input mode

Режим входных данных job: `plan_only` либо `estimate_and_plan`.

### Job

Один строительный кейс со своей папкой, manifest и артефактами.

### Job ID

Логическое безопасное имя кейса, заданное при `create_case`.

### LLM

Large Language Model — языковая модель. В скилле основная LLM ведёт процесс, а отдельные LLM-subagents выполняют узкие смысловые задачи.

### LLM context

Точный сохранённый набор данных, переданный аналитической модели. Хранится в `llm_context.json`.

### Manifest

Файл `manifest.json` с текущим состоянием job: режим входа, revision, подтверждение, hashes, статусы аудита и фотоэтапа.

### Mapping

Связь каждой строки сметы с каноническим помещением, видом работы и позицией каталога MCP.

### Mapping-subagent

Модельная задача для смыслового сопоставления, если exact mapping кодом невозможен.

### MCP

Внешний интерфейс, через который скилл получает поддерживаемые строительные работы и цены. В этом проекте используется `mcp_construction_prices__get_supported_works`.

### MCP price

Цена из сохранённой позиции каталога MCP. Валюта текущим контрактом не подтверждена.

### MCP work ID

Уникальный ID строительной работы в каталоге MCP, например `wall_painting`.

### Model lane

Выбранный runtime класс модели, например `main` или `light`. Это не обязательно имя конкретной модели или провайдера.

### Not found

Явный статус отсутствующего значения в ответе Vision. Означает, что значение не было надёжно найдено и его нельзя подставлять стандартом.

### Not checked

Проверка не выполнена из-за отсутствующих данных, несовместимого сопоставления или другой явно записанной причины.

### Observation

Наблюдаемая связь между переданными фактами в аналитическом insight. Не должна сама по себе утверждать причину.

### Orchestration / оркестрация

Управление порядком шагов: какой tool вызвать, когда запустить subagent, когда ждать пользователя и когда остановиться.

### Parent task

Основная задача, которая запустила дочерний subagent.

### Photo ID

Идентификатор импортированной фотографии. Используется, чтобы наблюдение нельзя было незаметно перенести на другой кадр.

### Photo token

Дополнительный секретный маркер связи между импортированной фотографией и её Vision-ответом.

### Photo Vision

Отдельная Vision-задача для одного кадра строительного объекта.

### Plan ID

Идентификатор плана внутри Vision contract. Python проверяет, что ответ относится к ожидаемому плану.

### Plan-only

Ветка без исходной сметы. После подтверждения геометрии система может сформировать предварительную XLSX-смету.

### Plugin

Python-модуль, который регистрирует публичные tools и связывает их с core-логикой.

### Projection

Проверенная копия изображения в специально разрешённой папке runtime. Vision получает именно её абсолютный путь, а не произвольное вложение.

### Provenance / происхождение данных

Информация о том, откуда взялось значение: файл, source row, сообщение пользователя, модельная задача, formula, revision и hash.

### Public tool

Операция скилла, которую может вызвать основная LLM. В текущей версии их четырнадцать.

### Raw result

Результат вычисления до отображаемого округления.

### Renderer / render-tool

Tool, который строит полный пользовательский Markdown из сохранённого состояния. Используется в том числе после handoff.

### Revision

Номер версии geometry. Каждая однозначная пользовательская correction создаёт новую revision.

### Room ID

Стабильный внутренний ID помещения вида `room_001`.

### Runtime

Среда, которая исполняет основную модель, tools и subagents, передаёт handoff и ведёт observability.

### Schema / схема

Машинное описание допустимой структуры данных: обязательные поля, типы, enum, диапазоны и запрет лишних полей.

### Schema validation

Проверка structured ответа по схеме до сохранения или расчёта.

### SHA-256

Криптографическая hash-функция. Используется для фиксации копий файлов и версий важных JSON-артефактов.

### Skipped

Позиция, которую система сознательно не создала или не проверила. Причина сохраняется явно, например `missing_geometry`.

### Source row

Исходный номер строки в Excel-смете. Позволяет связать finding и price check с конкретным местом документа.

### Source type

Тип источника измерения Vision: явная подпись, размерная линия, масштаб, производное значение, визуально обнаруженный символ или `not_found`.

### Staged attachment

Вложение, уже сохранённое runtime и описанное manifest entry, но ещё не импортированное в job.

### State machine / машина состояний

Правила допустимого порядка этапов. Например, аудит нельзя запустить до подтверждения geometry и сохранения каталога.

### Structured JSON

Ответ модели в заранее заданной объектной форме, который можно проверить программно. Не свободный Markdown.

### Subagent

Отдельная дочерняя model-задача с узкой целью, ограниченным context и запретами.

### Task ID

Идентификатор одного хода основной модели или дочерней задачи.

### Tool

Строго описанная операция, доступная модели. В данном скилле публичные tools реализованы на Python.

### Tool trace

Журнал вызовов tools, аргументов, статусов и результатов в observability/logs.

### Trace ID

Уникальный ID конкретной расчётной записи, например `room:room_001:net_wall_area_m2`.

### Unit compatibility

Проверка совместимости единиц. Например, работу в квадратных метрах нельзя сопоставить с контрольным количеством в штуках.

### Validator

Python-код, который принимает либо отклоняет входные данные по явным правилам.

### Vision / VLM

Модельная задача, способная смотреть изображение. В скилле используется для плана и фотографий, но её результат всегда проходит Python-валидацию.

### Visual insight

Проверенное наблюдение по конкретной фотографии. Не является доказательством состояния всего объекта.

### Wait task

Runtime-tool, который ожидает завершения subagent и возвращает его результат родителю.

### Warning

Предупреждение о неполноте, ограничении или несоответствии, которое не всегда блокирует весь процесс.

### XLSX

Формат Excel-файла. Технически является ZIP-контейнером с XML и другими внутренними файлами.

## Строительные и расчётные термины

### Валовая площадь стен

Периметр помещения, умноженный на высоту, до вычета дверей и окон.

### Геометрическая смежность

Факт, что дверь расположена на общей границе двух помещений. Не равен распределению монтажной работы в смете.

### Дверной проём

Отверстие в стене с шириной и высотой. Уменьшает чистую площадь стен; его ширина также вычитается из длины плинтуса.

### Контрольное значение

Количество или цена, независимо рассчитанные или полученные из проверенного источника для сравнения со сметой.

### Количество

Объём работы в её единице измерения: м², м или шт.

### Комплексная / ступенчатая форма

Помещение, которое нельзя надёжно описать одной длиной и шириной как прямоугольник. Требует дополнительных размеров или декомпозиции.

### Монтаж плинтуса

Работа, контрольный объём которой считается как периметр помещения минус ширины дверных проёмов.

### Оконный проём

Отверстие окна с шириной и высотой. Уменьшает чистую площадь стен.

### Отклонение

Разница между значением сметы и контролем. Может выражаться абсолютным числом и процентом.

### Периметр

Суммарная длина границ помещения.

### Площадь пола

Площадь помещения по плану или рассчитанная из подтверждённых размеров.

### Площадь потолка

В текущей модели принимается равной площади пола.

### Предварительный аудит

Автоматизированная проверка, которая выявляет данные для профессиональной перепроверки. Не окончательная экспертиза и не юридическое заключение.

### Смета

Таблица строительных работ по помещениям с единицами, количеством, ценой и стоимостью.

### Стоимость

Количество, умноженное на цену, с правилами округления кода.

### Уникальная дверь

Один физический дверной проём, даже если он записан в geometry двух смежных комнат с одинаковым `element_id`.

### Чистая площадь стен

Валовая площадь стен за вычетом площадей дверей и окон.

## Названия 14 публичных tools

### `create_case`

Создаёт job и manifest.

### `import_documents`

Импортирует и проверяет пару «смета + план».

### `import_plan`

Импортирует и проверяет план без исходной сметы.

### `save_geometry`

Валидирует Vision JSON и сохраняет geometry revision.

### `confirm_geometry`

Применяет correction либо подтверждает текущую revision.

### `render_geometry_review`

Строит полный Markdown review из сохранённой geometry.

### `save_price_catalog`

Валидирует и сохраняет каталог MCP.

### `generate_estimate`

После явного согласия пользователя создаёт эталонную (контрольную) XLSX-смету по подтверждённой геометрии и ценам MCP, а также JSON provenance.

### `run_audit`

Проверяет mapping и запускает детерминированные quantity/price checks.

### `skip_visual_review`

Фиксирует явный отказ пользователя от фотоэтапа.

### `import_site_photos`

Импортирует ZIP с фотографиями и готовит Photo Vision задачи.

### `save_visual_analysis`

Валидирует и объединяет результаты Photo Vision.

### `finalize_audit`

Валидирует аналитические insights и создаёт HTML.

### `render_audit_summary`

Строит полный итоговый Markdown из завершённого аудита.
