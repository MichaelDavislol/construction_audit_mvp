# Словарь терминов Construction Audit MVP

## Как пользоваться словарём

Здесь собраны пользовательские, строительные и технические термины, которые встречаются в скилле, трассировке и артефактах. Определения описывают именно смысл внутри Construction Audit MVP.

## A–Z и технические названия

### A2A

Сокращение от Agent-to-Agent. Так называется передача отдельной ограниченной задачи
от основной языковой модели субагенту. В проекте по этой схеме запускаются анализ
плана, смысловое сопоставление, анализ фотографий и формирование гипотез.

### Analyst / аналитический субагент

Отдельная модельная задача после программного аудита. Ищет связи между уже
сохранёнными результатами и формулирует проверяемые гипотезы. Не может изменять
расхождения, расчёты или наблюдения по фотографиям.

### Artifact / артефакт

Сохранённый результат этапа: JSON, XLSX, HTML или копия входного файла. Артефакт позволяет проверить, что именно участвовало в следующем шаге.

### Attachment / вложение

Файл, загруженный пользователем в сообщение. До импорта считается внешним входом и проходит проверку.

### Calculation trace / трассировка расчётов

Запись формулы, входных значений, результата, округления, помещения и предупреждений. Хранится в `calculation_trace.json`.

### Canonical / канонический

Приведённый к однозначному внутреннему виду. Например, название помещения
связывается со стабильным идентификатором `room_001`, а работа — с `mcp_work_id`.

### Chat ID

Идентификатор диалога. Объединяет несколько пользовательских сообщений и несколько задач основной модели.

### Confidence

Оценка уверенности модели распознавания в конкретном измерении. Не заменяет
основание измерения и не отменяет подтверждение пользователя.

### Context package

Пакет контекста: ограниченный сохранённый набор фактов для аналитического
субагента. Субагент не получает произвольные файлы и не должен добавлять сведения,
которых нет в пакете.

### Correction

Структурированное исправление помещения, двери или окна на основании явного
сообщения пользователя. Основная модель не редактирует весь JSON геометрии:
программа применяет только указанное изменение.

### Coverage / покрытие

Показатель полноты: сколько строк или цен проверено, сколько не проверено и почему. Нужен, чтобы частичный результат не выглядел полным.

### Decimal

Тип точной десятичной арифметики Python. Используется для объёмов, цен и процентов,
чтобы расчёты не зависели от арифметики языковой модели.

### Delegation packet

Пакет делегирования: готовое задание для субагента с ролью, целью, контекстом,
ограничениями, классом модели, режимом доступа и ожидаемым форматом JSON. Основная
модель передаёт пакет без пересказа и содержательных изменений.

### Deterministic / детерминированный

Выполняемый по фиксированному алгоритму. При одинаковых входных данных даёт
одинаковый результат. Так работают расчёты, точное сопоставление и программные
проверки входных данных.

### Evidence / доказательство

Конкретное основание значения или вывода: подпись на плане, строка сметы,
расхождение, проверка цены, запись расчёта или наблюдение по фотографии.

### Evidence ref

Структурированная ссылка аналитической гипотезы на существующий элемент пакета
контекста. Python проверяет наличие указанного элемента.

### Exact mapping

Точное сопоставление, которое Python выполняет без участия модели, если названия и
единицы совпадают однозначно.

### Fail closed

Принцип безопасной остановки: при недостатке или конфликте данных система не
угадывает, а останавливает зависимый этап или явно помечает проверку как
пропущенную либо невыполненную.

### Finding

Предварительное расхождение, найденное Python-кодом. Например, количество в смете
ниже контрольного значения или цена отличается от каталога MCP. Такое расхождение
не является доказанным нарушением.

### Geometry / геометрия

Структурированное описание помещений, размеров, площадей, высот, дверей, окон, источников и предупреждений.

### Geometry review

Обзор геометрии: полная таблица распознанных помещений, размеров, дверей и окон,
которую пользователь проверяет до подтверждения и расчётов.

### Handoff

Передача результата субагента обратно основной задаче через среду выполнения.
Может сопровождаться системным напоминанием и скрытым промежуточным ответом.

### Handoff barrier

Барьер передачи: обязательная остановка между сохранением результата субагента и
показом пользовательского Markdown. После неё средство формирования ответа заново
читает данные из сохранённого состояния.

### Hash

Хеш: цифровой отпечаток данных. В проекте используется алгоритм SHA-256.

### HTML escaping

Преобразование специальных символов перед вставкой текста в HTML. Не позволяет пользовательскому или внешнему тексту стать исполняемой разметкой.

### ID

Стабильный идентификатор сущности. Используются идентификаторы диалога, задачи,
запуска, помещения, фотографии, элемента, расхождения, гипотезы и расчётной записи.
В JSON их поля оканчиваются на `_id`.

### Insight

Аналитическая гипотеза, прошедшая программную проверку. Содержит наблюдение,
возможное объяснение, ограничения, ссылки на основания и рекомендуемую проверку.

### Input mode

Режим входных данных запуска: только план (`plan_only`) либо смета вместе с планом
(`estimate_and_plan`).

### Job

Отдельный запуск проверки со своим каталогом, манифестом и артефактами.

### Job ID

Логическое безопасное имя кейса, заданное при `create_case`.

### LLM

Сокращение от Large Language Model — большая языковая модель. Основная модель
управляет процессом, а отдельные субагенты выполняют узкие задачи распознавания и
смыслового анализа.

### LLM context

Точный сохранённый набор данных, переданный аналитической модели. Хранится в `llm_context.json`.

### Manifest

Файл `manifest.json` с текущим состоянием запуска: режимом входных данных, версией
геометрии, подтверждением, хешами и статусами этапов.

### Mapping

Связь каждой строки сметы с каноническим помещением, видом работы и позицией каталога MCP.

### Mapping-subagent

Субагент смыслового сопоставления. Используется, когда Python не может однозначно
сопоставить названия и требуется понять их смысл.

### MCP

Внешний интерфейс, через который скилл получает поддерживаемые строительные работы и цены. В этом проекте используется `mcp_construction_prices__get_supported_works`.

### MCP price

Цена из сохранённой позиции каталога MCP. Текущая схема каталога не подтверждает
валюту.

### MCP work ID

Уникальный идентификатор строительной работы в каталоге MCP, например
`wall_painting`.

### Model lane

Класс модели, выбранный средой выполнения, например `main` или `light`. Это не
обязательно название конкретной модели или поставщика.

### Not found

Статус `not_found` означает, что модель распознавания не нашла значение с
достаточной надёжностью. Вместо него нельзя подставлять типовое значение.

### Not checked

Проверка не выполнена из-за отсутствующих данных, несовместимого сопоставления или другой явно записанной причины.

### Observation

Наблюдаемая связь между переданными фактами внутри аналитической гипотезы. Сама по
себе не устанавливает причину.

### Orchestration / оркестрация

Управление порядком шагов: какую функцию вызвать, когда запустить субагента, когда
ждать ответа пользователя и когда остановить процесс.

### Parent task

Родительская задача, которая запустила субагента.

### Photo ID

Идентификатор импортированной фотографии. Используется, чтобы наблюдение нельзя было незаметно перенести на другой кадр.

### Photo token

Случайный внутренний маркер, связывающий импортированную фотографию с ответом
модели распознавания. Не предназначен для публикации.

### Photo Vision

Отдельная задача визуального анализа одной фотографии строительного объекта.

### Plan ID

Идентификатор плана в контракте визуального анализа. Python проверяет, что ответ
относится к ожидаемому изображению.

### Plan-only

Ветка без исходной сметы. После подтверждения геометрии система может сформировать предварительную XLSX-смету.

### Plugin

Модуль Python, который регистрирует публичные функции скилла и связывает их с
основной логикой.

### Projection

Проверенная копия изображения в специально разрешённом каталоге среды выполнения.
Модель визуального анализа получает эту копию, а не произвольное вложение.

### Provenance / происхождение данных

Информация о происхождении значения: исходный файл или строка, сообщение
пользователя, модельная задача, формула, версия данных и хеш.

### Public tool

Публичная функция скилла, которую может вызвать основная языковая модель. В текущей
версии таких функций четырнадцать.

### Raw result

Результат вычисления до отображаемого округления.

### Renderer / render-tool

Средство формирования ответа: функция, которая строит полный пользовательский
Markdown из сохранённого состояния, в том числе после возврата результата
субагента.

### Revision

Номер версии геометрии. Каждое однозначное исправление пользователя создаёт новую
версию.

### Room ID

Стабильный внутренний ID помещения вида `room_001`.

### Runtime

Среда выполнения: система, которая запускает основную модель, публичные функции и
субагентов, передаёт результаты и ведёт технический журнал.

### Schema / схема

Машинное описание допустимой структуры данных: обязательные поля, типы,
перечисления, диапазоны и правила для дополнительных полей.

### Schema validation

Проверка структурированного ответа по схеме перед сохранением или расчётом.

### SHA-256

Криптографическая хеш-функция. Используется для контроля копий файлов и версий
важных JSON-артефактов.

### Skipped

Позиция, которую система сознательно не создала или не проверила. Причина сохраняется явно, например `missing_geometry`.

### Source row

Номер строки в исходной Excel-смете. Позволяет связать расхождение и проверку цены
с конкретным местом документа.

### Source type

Тип источника измерения: явная подпись, размерная линия, масштаб, производное
значение, визуально обнаруженный символ или статус `not_found`.

### Staged attachment

Подготовленное вложение: файл, который среда выполнения уже сохранила и внесла в
манифест, но скилл ещё не импортировал в запуск.

### State machine / машина состояний

Правила допустимого порядка этапов. Например, аудит нельзя запустить до
подтверждения геометрии и сохранения каталога цен.

### Structured JSON

Ответ модели в заранее заданной объектной форме, который можно проверить
программно. Вместо произвольного текста модель возвращает поля и значения по
заданной схеме.

### Subagent

Субагент: отдельная модельная задача с узкой целью, ограниченным контекстом и
явными запретами.

### Task ID

Идентификатор одного хода основной модели или дочерней задачи.

### Tool

Строго описанная операция, доступная модели. Публичные функции этого скилла
реализованы на Python.

### Tool trace

Журнал вызовов функций: их аргументы, статусы и результаты в техническом журнале
среды выполнения.

### Trace ID

Уникальный ID конкретной расчётной записи, например `room:room_001:net_wall_area_m2`.

### Unit compatibility

Проверка совместимости единиц. Например, работу в квадратных метрах нельзя сопоставить с контрольным количеством в штуках.

### Validator

Программная проверка, которая принимает или отклоняет входные данные по явным
правилам.

### Vision / VLM

Модель с возможностью анализа изображений. Используется для плана и фотографий;
её структурированный ответ всегда проходит программную проверку.

### Visual insight

Проверенное наблюдение по конкретной фотографии. Не является доказательством состояния всего объекта.

### Wait task

Операция среды выполнения, которая ожидает завершения субагента и возвращает его
результат родительской задаче.

### Warning

Предупреждение о неполноте, ограничении или несоответствии, которое не всегда блокирует весь процесс.

### XLSX

Формат Excel-файла. Технически является ZIP-контейнером с XML и другими внутренними файлами.

## Строительные и расчётные термины

### Валовая площадь стен

Периметр помещения, умноженный на высоту, до вычета дверей и окон.

### Геометрическая смежность

Факт, что дверь расположена на общей границе двух помещений. Не означает, что
монтаж этой двери отнесён в смете к конкретному помещению.

### Дверной проём

Отверстие в стене с шириной и высотой. Уменьшает чистую площадь стен; его ширина также вычитается из длины плинтуса.

### Контрольное значение

Количество или цена, независимо рассчитанные или полученные из проверенного источника для сравнения со сметой.

### Количество

Объём работы в её единице измерения: м², м или шт.

### Комплексная / ступенчатая форма

Помещение, которое нельзя надёжно описать одной длиной и шириной как прямоугольник. Требует дополнительных размеров или декомпозиции.

### Монтаж плинтуса

Работа, контрольный объём которой считается как периметр помещения минус ширины дверных проёмов.

### Оконный проём

Отверстие окна с шириной и высотой. Уменьшает чистую площадь стен.

### Отклонение

Разница между значением сметы и контролем. Может выражаться абсолютным числом и процентом.

### Периметр

Суммарная длина границ помещения.

### Площадь пола

Площадь помещения по плану или рассчитанная из подтверждённых размеров.

### Площадь потолка

В текущей модели принимается равной площади пола.

### Предварительный аудит

Автоматизированная проверка, которая выявляет данные для профессиональной перепроверки. Не окончательная экспертиза и не юридическое заключение.

### Смета

Таблица строительных работ по помещениям с единицами, количеством, ценой и стоимостью.

### Стоимость

Количество, умноженное на цену за единицу. Округление выполняется по правилам,
заданным в программе.

### Уникальная дверь

Один физический дверной проём, даже если он записан в геометрии двух смежных
помещений с одинаковым `element_id`.

### Чистая площадь стен

Валовая площадь стен за вычетом площадей дверей и окон.

## Названия 14 публичных функций

### `create_case`

Создаёт отдельный запуск и его манифест.

### `import_documents`

Импортирует и проверяет пару «смета + план».

### `import_plan`

Импортирует и проверяет план без исходной сметы.

### `save_geometry`

Проверяет JSON визуального анализа и сохраняет новую версию геометрии.

### `confirm_geometry`

Применяет исправление либо подтверждает текущую версию геометрии.

### `render_geometry_review`

Строит полный обзор сохранённой геометрии в формате Markdown.

### `save_price_catalog`

Проверяет и сохраняет каталог MCP.

### `generate_estimate`

После явного согласия пользователя создаёт контрольную XLSX-смету по
подтверждённой геометрии и ценам MCP, а также JSON с данными о происхождении.

### `run_audit`

Проверяет сопоставление и запускает программные проверки количества и цены.

### `skip_visual_review`

Фиксирует явный отказ пользователя от фотоэтапа.

### `import_site_photos`

Импортирует ZIP с фотографиями и готовит отдельную задачу визуального анализа для
каждого кадра.

### `save_visual_analysis`

Проверяет и объединяет результаты визуального анализа фотографий.

### `finalize_audit`

Проверяет аналитические гипотезы и создаёт HTML-отчёт.

### `render_audit_summary`

Строит полный итоговый Markdown из завершённого аудита.
