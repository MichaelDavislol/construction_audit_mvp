# Сценарий 01: квартира с намеренными ошибками

Фактический запуск `audit_6_20260718` от 18 июля 2026 года, версия скилла `0.7.7`. В исходную смету заранее внесены ошибки; ожидаемый результат — обнаружить их и завершить всю проверку без потери полноты.

**Коротко:** в смете специально изменили объёмы и цены. Скилл проверил все 21 строку, нашёл четыре связанных с этим замечания и не остановился из-за ошибочных данных.

## Вход

- [`plan.png`](input/plan.png) — план квартиры из трёх помещений;
- [`estimate.xlsx`](input/estimate.xlsx) — смета на 21 строку с намеренными ошибками.

## Фактический результат

| Метрика | Значение |
|---|---:|
| Версия геометрии | 2, подтверждена |
| Помещений / площадь пола | 3 / 123,16 м² |
| Уникальных дверей / окон | 2 / 3 |
| Строк сметы | 21 |
| Полнота проверки объёмов / цен | 100% / 100% |
| Quantity deviations | 1 |
| Price deviations | 2 |
| Расхождения | 4 |
| High / warning | 2 / 2 |
| Гипотезы аналитического этапа | 2 |
| Строк контрольной сметы | 19 |

Скилл зафиксировал арифметические несоответствия в строках 2 и 11, заниженный объём устройства пола в строке 11 (`24,16` против `40,56 м²`) и завышенную цену окраски стен в строке 2 (`560` против `450` за м²).

Машиночитаемая сводка: [`result-summary.json`](result-summary.json). Полный пользовательский результат: [`report.html`](output/report.html).

## Что показывает кейс

Заложенные ошибки не мешают процессу завершиться: все 21 строка проверены, расхождения разделены по типам и важности, расчётные значения связаны с трассировкой, а аналитический этап формулирует гипотезы на основе уже сохранённых детерминированных результатов.

Кейс воспроизводим при повторном запуске на тех же входных документах: намеренно изменённые значения находятся в самой XLSX-смете, а не возникают из случайной ошибки Vision.

В [`output/`](output/) находятся все 16 файлов фактического каталога результатов, включая нормализованную смету, проверку и исправления геометрии, сопоставление, каталог MCP, рассчитанные объёмы, трассировку вычислений, проверки цен, расхождения, аналитические артефакты, HTML и контрольную XLSX.
