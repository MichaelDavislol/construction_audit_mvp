# Case 10: смета только по плану с недостающими размерами

Фактический запуск `audit_medoffice_20260719b` от 19 июля 2026 года, версия скилла `0.7.7`. На вход передан только план медицинского офиса: исходной XLSX нет. На изображении намеренно стёрты все линейные размеры помещения «Ресепшен».

Это тот же файл плана, который использован в [Case 08](../08-missing-room-dimensions-user-correction/README.md), но здесь проверяется отдельная ветка `plan_only`: после исправления geometry скилл должен самостоятельно получить цены MCP и построить полную смету.

**Коротко:** исходной сметы вообще нет, а размеры одного помещения стёрты с плана. Скилл запросил недостающие данные, пересчитал помещение и создал с нуля полную смету на 31 строку без пропусков.

## Вход

- [`plan.png`](input/plan.png) — единственный входной документ;
- исходная смета отсутствует, `input_mode: plan_only`.

![План со стёртыми размерами ресепшена](input/plan.png)

## Запрос недостающих данных

Vision распознал пять помещений, двери и окна, но не восстановил стёртые размеры ресепшена догадкой. В geometry была сохранена явная проблема: `Нет подписанных линейных размеров/площади для этого помещения на плане`.

Пользователь передал correction:

> Ресепшн размеры ширина 4.2 длина 9.2. Высота всех помещений 2.8 метра

Скилл создал geometry revision 2 и зафиксировал происхождение значений:

| Поле | Значение | Source type |
|---|---:|---|
| Ширина ресепшена | 4,2 м | `user_correction` |
| Длина ресепшена | 9,2 м | `user_correction` |
| Высота пяти помещений | 2,8 м | `user_correction` |
| Площадь ресепшена | 38,64 м² | `derived_from_explicit_dimensions` |
| Периметр ресепшена | 26,8 м | `derived_from_explicit_dimensions` |

После пользовательского подтверждения revision 2 стала единственным основанием дальнейших вычислений.

## Расчёт новой сметы только по плану

Без исходной XLSX скилл получил через MCP каталог из семи поддерживаемых работ и построил [`generated_estimate.xlsx`](output/generated_estimate.xlsx) непосредственно по подтверждённой geometry.

Для ресепшена рассчитаны:

| Работа / показатель | Количество |
|---|---:|
| Грунтовка стен | 67,12 м² |
| Окраска стен | 67,12 м² |
| Устройство пола | 38,64 м² |
| Отделка потолка | 38,64 м² |
| Монтаж плинтуса | 24,10 м |
| Установка окон | 1 шт. |

Отдельная объектная строка содержит 6 уникальных дверей по цене `7 000`, итого `42 000`.

Plan-only workflow не создаёт аудит исходной сметы, findings или HTML-отчёт, потому что проверять нечего. Его результат — подтверждённая geometry, каталог MCP и новая расчётная XLSX.

Машиночитаемая сводка: [`result-summary.json`](result-summary.json). В [`output/`](output/) находятся все шесть содержательных файлов результата. Временный lock-файл Excel `~$generated_estimate.xlsx` не включён: он создан приложением просмотра и не является артефактом скилла.
