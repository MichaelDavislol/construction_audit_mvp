# Демонстрация: предварительная смета только по плану

Это фактический запуск `plan_med_20260718` от 18 июля 2026 года. В примере сохранён реальный процесс работы, а не эталонный тест побайтового совпадения.

## Вход

- [`input/plan.png`](input/plan.png) — план медицинского офиса из пяти помещений;
- исходная XLSX отсутствует.

## Фактический порядок работы

1. Vision распознал пять помещений общей площадью `164,36 м²`, шесть уникальных дверей и пять окон.
2. Высота помещений отсутствовала на плане и осталась неопределённой — скилл не подставил типовое значение.
3. Пользователь явно сообщил: «Высота всех помещений 2.8 метра».
4. Python применил ограниченное исправление ко всем пяти помещениям и создал ревизию геометрии 2.
5. Пользователь отдельным сообщением подтвердил новую ревизию.
6. Скилл получил и валидировал семь позиций каталога `price-mcp`.
7. После отдельного согласия пользователя была создана предварительная XLSX-смета на 31 строку.

## Что показывает пример

Ветка только с планом не создаёт фиктивную исходную смету и не проводит сравнение с несуществующим документом. Она сначала требует подтверждённую геометрию, затем получает внешний каталог и только после отдельного согласия создаёт новую контрольную XLSX.
