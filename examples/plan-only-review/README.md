# Demo: plan-only с генерацией предварительной сметы

Фактический запуск `plan_med_20260718` от 18 июля 2026 года, версия скилла `0.7.6`. Публикуемый payload имеет версию `0.7.7`, поэтому пример демонстрирует реальный workflow, а не golden test побайтового совпадения.

## Вход

- [`input/plan.png`](input/plan.png) — план медицинского офиса из пяти помещений;
- исходная XLSX отсутствует.

## Фактический workflow

1. Vision распознал пять помещений общей площадью `164,36 м²`, шесть уникальных дверей и пять окон.
2. Высота помещений отсутствовала на плане и осталась missing — скилл не подставил типовое значение.
3. Пользователь явно сообщил: «Высота всех помещений 2.8 метра».
4. Python применил ограниченную correction ко всем пяти помещениям и создал geometry revision 2.
5. Пользователь отдельным сообщением подтвердил новую revision.
6. Скилл получил и валидировал семь позиций каталога `price-mcp`.
7. После отдельного согласия пользователя была создана предварительная XLSX-смета на 31 строку.

## Артефакты

В [`output/`](output/) находятся все шесть фактических артефактов plan-only job:

| Файл | Назначение |
|---|---|
| [`geometry.json`](output/geometry.json) | Canonical geometry revision 2 |
| [`geometry_review.json`](output/geometry_review.json) | Structured review подтверждённой geometry |
| [`geometry_corrections.json`](output/geometry_corrections.json) | История явной correction высоты |
| [`price_catalog.json`](output/price_catalog.json) | Валидированный каталог из семи MCP-позиций |
| [`generated_estimate.json`](output/generated_estimate.json) | Строки сметы, coverage, hashes и provenance |
| [`generated_estimate.xlsx`](output/generated_estimate.xlsx) | Готовая предварительная XLSX-смета |

## Что показывает пример

Plan-only ветка не создаёт фиктивную исходную смету и не проводит сравнение с несуществующим документом. Она сначала требует подтверждённую геометрию, затем получает внешний каталог и только после отдельного согласия создаёт новую контрольную XLSX.
