# Демонстрация: полный аудит медицинского офиса

Это обезличенный фактический запуск `audit_med_20260717`, выполненный 17–18 июля 2026 года с версией скилла `0.7.5`. Публикуемый пакет имеет версию `0.7.7`, поэтому пример демонстрирует реальный процесс, а не эталонный тест побайтового совпадения.

## Вход

- [`plan.png`](input/plan.png) — план пяти помещений;
- [`estimate.xlsx`](input/estimate.xlsx) — смета на 35 строк;
- [`site-photos.zip`](input/site-photos.zip) — архив из двух фотографий;
- `input/site-photos/` — те же фотографии отдельно для просмотра на GitHub.

Исходные материалы не содержат ФИО, адреса или контактных данных. Имена загрузок, локальные пути, task IDs и delegation tokens не включены в demo metadata.

## Как повторить

1. Установить скилл и запустить MCP по корневому `INSTALL.md`.
2. В новом диалоге загрузить `plan.png` и `estimate.xlsx`.
3. Попросить провести предварительный аудит сметы по плану.
4. Проверить обзор геометрии. В фактическом запуске пользователь указал высоту всех помещений `2,8 м`, после чего была создана ревизия 2.
5. Отдельным сообщением подтвердить ревизию.
6. После детерминированного аудита загрузить `site-photos.zip`.
7. Дождаться Photo Vision, Analyst и формирования HTML.
8. После отчёта принять предложение скилла создать эталонную (контрольную) XLSX-смету.

## Готовые артефакты

В [`output/`](output/) включены все 18 файлов фактического выходного каталога: автономный HTML, контрольная XLSX, нормализованная смета, геометрия и исправления, каталог цен, сопоставление строк, количества, трассировка расчётов, проверки цен, найденные расхождения, анализ фотографий и аналитический этап.

Назначение каждого файла и порядок их появления описаны в [`ARTIFACTS.md`](ARTIFACTS.md).

HTML содержит исходные имена подготовленных файлов и технические ID задач, сохранённые генератором отчёта для подтверждения происхождения данных. Он не содержит локальных абсолютных путей, токенов делегации или секретов.

Два JSON были минимально санитизированы для публичной публикации: в `mapping.json` заменён токен делегации, а в `visual_photos.json` заменены два токена делегации и два абсолютных локальных пути. Остальные выходные файлы перенесены без изменений. Подробности и исходные SHA-256 указаны в [`SANITIZATION.md`](SANITIZATION.md).

## Ограничение интерпретации

Findings и гипотезы являются предварительными. Demo показывает работу конвейера и воспроизводимость расчётов, но не является заключением строительного эксперта.
